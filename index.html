Stakeholder Project:

Horticultural Database Integration

Python Pandas AirTable Data Cleaning
Data Engineering Data Modeling

I transformed and loaded scientific plant information to enrich an inventory database.

view project details

My Role:
Volunteer Data Specialist for the Wave Hill Horticulture Department

Context:
Wave Hill is a non-profit public garden with a Horticultural Inventory database for internal use. Taxonomic information for each database record, such as the native range of a plant, was entered individually based on manual internet lookups.

Goal:
I was asked to eliminate the tedious internet lookups by integrating scientific plant information directly in the existing Horticultural Inventory database.

Data Description:
Source Dataset Data Format Size (rows)
Plant records extracted from
the Horticultural Inventory database (DB)		 CSV file ~1,300
Plant information from
the U.S. Dept. of Agriculture (USDA)		 CSV file ~93,000
Taxonomic information from
the World Checklist of Vascular Plants (WCVP)		 CSV file ~1.4 million

Key Challenge:
Each of the data sources handles the scientific plant names differently. Cleaning and aligning the scientific names was crucial to joining the 3 datasets.

Scientific Plant Name
Species Name Infraspecies Rank
Genus Specific Epithet Infraspecies Designation Infraspecies Name

What I Built:
I used Python code to parse scientific plant names into a standard format and join the source datasets. The joined data was loaded into the Horticultural Inventory database as a new table.

  1. I wrote a Python module to deconstruct a scientific plant name into taxonomic nomenclature components: genus, specific epithet, hybrid, and infraspecies. My code handles common abbreviation variations, such as subsp. or ssp for subspecies.

  2. I loaded the DB and USDA datasets into Pandas dataframes, then I parsed their scientific name columns using the parser module from step 1.

  3. I joined the DB, USDA, and WCVP dataframes using the parsed scientific plant name components as a composite key.

Deliverables:
  • A CSV file of combined USDA and WCVP taxonomic information corresponding to the 1310 unique plant names in the Horticultural Inventory database. This CSV was loaded as a new table in the database. The scientific plant name is the foreign key that allows the taxonomic information to be joined with the original inventory table.

  • A Python notebook that connects to the database through an API so users can easily add new rows of taxonomic information.

Impact:
  • Taxonomic information was matched for 96% of the plant records in the Horticultural Inventory. This saved 7 hours of manual lookups of the same information, assuming 20 seconds per lookup.

  • Data validation - any unmatched records indiciate potential spelling errors or unaccepted scientific names in the original inventory table.

  • Rapid future loading of taxonomic information to the database via the API-connected notebook, which also avoids the copy-and-paste errors of manual data entry.

Stakeholder Project:

Slipcase Price Tuning

Python Pandas

I created a visualization tool to help a box manufacturer adjust their pricing.

view project details

My Role:
Freelance work for a fine-art shipper

Context:
A manufacturer of custom art-packing boxes (slipcases) was using a simple linear function to calculate retail prices, but this did not scale well for extreme sizes. The prices felt too high for small boxes and too low for large boxes. The manufacturer wanted to avoid adding a separate surcharge for large boxes.

Goal:
I was asked to produce a single, continuous price function that performed better at the extreme ends of the product range.

Key Challenge:
Creating an easy-to-grasp visualization that shows changes to the price function.

What I Built:
I wrote a new, adjustable price function with exponential expressions that target specific box sizes. I also created a grid visualization that shows updated prices when adjustments are made to the function.

This is an image of the visualization after price adjustments for small and large boxes (box length and height in inches are shown along the axes.). The left grid contains final retail price ($), and the right grid contains changes in price from the base price model. Note the large field of 0's and 1's in the middle of the right grid, indicating only a minor impact to the underlying prices.

  • Multiple coefficients can be adjusted to independently target quadrants in the price grid. Prices can be throttled or increased, but the base price model, for example $.55 per linear inch, remains for non-adjusted box sizes.

  • The visualization script runs in a Python notebook. The manufacturer can experiment with different combinations until the prices of small and large boxes feel right without affecting the midsize box pricing. The notebook outputs the corresponding 'tuned' price function, which the manufacturer can copy and paste into the business software of their choice.

Deliverables:
  • A Python notebook that the manufacturer can run to fine tune their product prices.

Impact:
  • The manufacturer was able to easily adjust their prices. Small custom boxes are more appealing to price-conscious customers, and large boxes provide a proper profit margin for the manufacturer.

Independent Project:

Squirrel Census EDA

Python Pandas Numpy
Matplotlib Data Visualization Data Transformation

My exploratory data analysis of squirrel behavior observed in the 2018 Central Park Squirrel Census.

view project details

Context:
The 2018 Central Park Squirrel Census project was conducted by volunteers and organizations throughout the park's 843 acres and various use areas. Squirrel locations, characteristics, and behaviors were recorded over several days.

Goal:
Explore the Squirrel Census data and describe the major squirrel behaviors. Would squirrel behaviors change across different parts of the park?

Data Description:
I obtained the 3023-row Squirrel Census data through the NYC Open Data api. Each row details a squirrel sighting with coordinates, a hectare grid location, physical description, standardized behaviors, and subjective behavioral observations.

Techniques:
I explored the data in a Jupyter notebook. A quick plot of coordinates shows the distribution of squirrels in the park. The large, central blank area indicates the water reservoir.

The census records major behaviors for each squirrel. I tallied the squirrel behaviors and grouped them into 3 behavioral categories.

To get a picture of behavioral patterns throughout the park, I parsed the hectare id for each row, transformed the shape of the data, and aggregated behaviors for the individual hectares. I then plotted the aggreations as heat maps.

I also divided the park into 8 cross sections for further aggregation (indiciated in orange in the "Squirrels per Hectare" heat map above). Each of these lateral cross sections holds 45 or 54 hectares.

Key Findings:

Squirrel behaviors change as the park topography and use areas change. As seen in the heat maps above, feeding behavior is widespread while communication behavior is more sporadic and observed more in the center of the park (cross sections 4-6).

The individual squirrel behaviors can be aggregated per park cross section, plotted from south to north, and checked for correlation. In general, foraging and eating behaviors are correlated, and they are negatively correlated with communication behaviors. This is most evident in the center of the park (cross sections 4-6) where feeding behavior decreases markedly.

Further research would look at the influence of changes in human presence on squirrel behavior.
